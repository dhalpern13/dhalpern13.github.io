# Project: Daniel Halpern's Personal Website

## Overview
This is a Jekyll-based GitHub Pages personal academic website for Daniel Halpern (dhalpern13.github.io).
The live site is at **https://daniel-halpern.com**.

## Build & Compile

- **Use `uv run` for all Python commands** (e.g., `uv run python compile-data.py`). Do NOT run Python directly or activate the venv manually.
- **`uv run python compile-data.py`**: The primary build script. It:
  1. Reads publication data from `_data/publication-data.yml`
  2. Generates YAML data files for the website (in `_data/`)
  3. Generates LaTeX `.tex` files for the resume (in `_resume/`)
  4. Compiles the resume PDF via `pdflatex` (runs 3 times for references)
  5. Copies the compiled PDF to `files/resume.pdf`
  - Pass any argument (e.g., `python compile-data.py --no-latex`) to skip LaTeX compilation and only regenerate data files.

- **Do NOT run `pdflatex` directly.** Always use `compile-data.py` so the generated `.tex` files stay in sync with `publication-data.yml`.

## Directory Structure

- `_resume/` — LaTeX source files (resume.tex, publications.tex, journal.tex, working.tex, scimisc-cv.sty). **Not served by Jekyll** (underscore prefix).
- `files/` — Publicly served files: resume.pdf and paper PDFs.
- `_data/` — YAML data files consumed by Jekyll templates and generated by `compile-data.py`.
- `_pages/`, `_layouts/`, `_includes/`, `_sass/` — Standard Jekyll directories.
- `images/` — Site images.
